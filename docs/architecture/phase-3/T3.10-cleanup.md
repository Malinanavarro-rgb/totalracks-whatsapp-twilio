# T3.10 — Cleanup: `render.yaml` y `.env.example`

**Estado:** Pendiente  
**Tipo:** Configuración  
**Archivos:** `render.yaml`, `.env.example`  
**Riesgo:** Ninguno — solo documentación de configuración, no afecta comportamiento

---

## Objetivo

Actualizar los archivos de configuración para reflejar la realidad post-FASE 3. `COMPANY_SLUG` ya no es necesario para el routing; mantenerlo como variable requerida en `render.yaml` es engañoso para cualquier desarrollador que trabaje en el proyecto en el futuro.

---

## Por qué hacer este cleanup

El código es documentación. Si `render.yaml` dice que `COMPANY_SLUG` es necesario, un desarrollador nuevo asumirá que el sistema lo usa para algo crítico y tendrá miedo de quitarlo. Un `.env.example` desactualizado crea confusión sobre qué variables son realmente necesarias.

Esta tarea mantiene la coherencia entre el código, la configuración y la realidad del sistema.

---

## Cambios en `render.yaml`

```yaml
# Antes:
- key: COMPANY_SLUG
  value: totalracks

# Después: (variable eliminada o comentada con explicación)
# COMPANY_SLUG fue removido en FASE 3.
# El routing ahora es dinámico via tabla channel_endpoints en Supabase.
# Ver docs/architecture/phase-3/README.md
```

**Nota:** Si Render ya tiene `COMPANY_SLUG` como variable de entorno activa, no hace falta eliminarla del dashboard de Render. El código ya no la usa, así que tenerla no causa ningún problema. Solo se elimina del `render.yaml` para que futuros deploys no la incluyan.

---

## Cambios en `.env.example`

```bash
# Antes:
# ── Empresa ───────────────────────────────────────────────────────────────────
# Slug que identifica a la empresa en Supabase (tabla companies)
COMPANY_SLUG=totalracks

# Después: sección actualizada
# ── Routing ────────────────────────────────────────────────────────────────────
# A partir de FASE 3, el routing es dinámico. No se necesita COMPANY_SLUG.
# Para agregar una empresa: insertar fila en tabla channel_endpoints en Supabase.
# Ver: docs/architecture/phase-3/T3.1-channel-endpoints.md
```
