# T3.2 — Sembrar número de Total Racks en `channel_endpoints`

**Estado:** Pendiente  
**Tipo:** DB seed  
**Riesgo:** Bajo — INSERT en tabla nueva, rollback es un DELETE

---

## Objetivo

Insertar el número de Twilio de Total Racks en `channel_endpoints` para que, cuando el nuevo routing esté activo, los mensajes existentes sigan funcionando exactamente igual.

---

## Por qué se hace antes que los cambios de código

Si el código del Channel Router entra antes de que exista la fila de Total Racks en `channel_endpoints`, todos los mensajes de WhatsApp de Total Racks comenzarían a recibir TwiML vacío (número no reconocido). Esto cortaría producción.

El orden correcto es: datos primero, código después. Cuando el servidor empiece a enrutar por esta tabla, la fila ya debe existir.

---

## SQL de inserción

Requiere el UUID de Total Racks en la tabla `companies`. Obtenerlo primero:

```sql
SELECT id, slug, nombre FROM companies WHERE slug = 'totalracks';
```

Luego insertar con el UUID real:

```sql
INSERT INTO channel_endpoints (company_id, endpoint, canal, activo)
VALUES (
  '<UUID_DE_TOTALRACKS>',           -- resultado de la query anterior
  'whatsapp:+14155238886',          -- sandbox Twilio (actualizar al número real de producción)
  'whatsapp',
  true
);
```

**Importante:** El `endpoint` debe ser el número exacto tal como aparece en `req.body.To` del webhook de Twilio, incluyendo el prefijo `whatsapp:`. Verificar en los logs de Render o en la consola de Twilio cuál es el formato exacto.

---

## Cuándo hay dos números (sandbox + producción)

Si Total Racks tiene tanto el número sandbox (`whatsapp:+14155238886`) como un número productivo real, insertar ambos apuntando al mismo `company_id`:

```sql
INSERT INTO channel_endpoints (company_id, endpoint, canal, activo) VALUES
  ('<UUID>', 'whatsapp:+14155238886',    'whatsapp', true),  -- sandbox
  ('<UUID>', 'whatsapp:+521XXXXXXXXXX',  'whatsapp', true);  -- producción
```

---

## Verificación

```sql
SELECT e.endpoint, e.canal, e.activo, c.nombre
FROM channel_endpoints e
JOIN companies c ON e.company_id = c.id;
```

Resultado esperado: al menos una fila con `nombre = 'Total Racks'` (o el nombre exacto en la tabla `companies`).
