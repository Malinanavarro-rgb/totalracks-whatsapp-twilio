## Objetivo

Conectar el Channel Router al webhook de Twilio. A partir de este momento, el servidor determina qué empresa atiende cada mensaje consultando `channel_endpoints` en lugar de leer `COMPANY_SLUG`.

---

## Por qué este es "el corte"

Todas las tareas anteriores (T3.1 a T3.8) pueden aplicarse sin cambiar el comportamiento de producción. Los datos de DB existen pero no se usan para routing. El código del router existe pero nadie lo llama. Los módulos refactorizados funcionan cuando se prueban localmente.

T3.9 es el momento donde todo se conecta. Después de este commit + deploy, Total Racks recibe mensajes gracias a la fila en `channel_endpoints`, no gracias a `COMPANY_SLUG`. Si esa fila no existe o tiene el número incorrecto, Total Racks deja de responder.

**Prerequisito obligatorio:** T3.2 (número de Total Racks en `channel_endpoints`) debe estar correctamente ejecutado antes de este deploy.

## Cambios en el webhook handler

**Antes:**
```js
app.post('/webhook/twilio', async (req, res) => {
  // ...
  const message   = adapter.parseIncoming(req);
  const resultado = await orchestrator.procesarMensaje(message);
  // ...
});
```

**Después:**
```js
app.post('/webhook/twilio', async (req, res) => {
  // ...
  const message = adapter.parseIncoming(req);

  // FASE 3: routing dinámico por número receptor
  const routeResult = await channelRouter.enrutar(message.incoming_endpoint);
  if (!routeResult) {
    // Número no registrado — silencio controlado
    console.warn('⚠️  Endpoint sin empresa registrada:', message.incoming_endpoint);
    return res.type('text/xml').send('<Response></Response>');
  }
  message.company_id = routeResult.company_id;

  const resultado = await orchestrator.procesarMensaje(message);
  // ...
});
```

## Rollback (emergencia)

Si Total Racks deja de responder después del deploy:

```bash
# 1. Revertir el código
git revert <commit-hash>
git push origin main
# Render redeploya automáticamente con el código anterior

# 2. Si el problema es en los datos (número incorrecto en channel_endpoints):
# Corregir el endpoint en Supabase SQL Editor:
UPDATE channel_endpoints
SET endpoint = 'whatsapp:+NUMERO_CORRECTO'
WHERE company_id = '<UUID_TOTALRACKS>';
# El caché del router expira en 5 min o se fuerza con redeploy
```

**Tiempo estimado de rollback total:** menos de 3 minutos.
